# Why Skills Are Not Enough: Principles and the Generative Hierarchy

> **Source-of-truth:** [PD.FORM.047](https://github.com/aisystant/PACK-personal/blob/main/pack/personal-development/02-domain-entities/formalizations/PD.FORM.047-generative-principles-hierarchy.md)

## Core Idea

Skills are a necessary but **insufficient** level of Mastery. They sit at the lowest level of a four-level hierarchy of principles and are **generated by** those principles — not a replacement for them.

## Four Levels

| Level | What | Why an agent needs it | Without it |
|-------|------|-----------------------|------------|
| **Level 0: ZP** | Zero principles (transdisciplines) | Ability to navigate any new Domain | Every domain change means starting from scratch |
| **Level 1: FPF** | First principles (correctness frame) | Precise language: Role ≠ Method ≠ Work ≠ Capability | Everything is called a "process" — confusion |
| **Level 2: Pack** | Second principles (domain knowledge) | Source-of-truth: distinctions, methods, failure modes | Tribal knowledge; every project starts over |
| **Level 3: DS** | Skills, programs, tools | Concrete execution | Nothing to do |

**Fallback chain:** DS → Pack → SPF → FPF → ZP. If something is unclear at the DS level, move up to Pack. If Pack does not help, move to FPF. Continue up to ZP.

## Four Arguments: Why Skills Are Not Enough

### 1. A Skill Is Procedural, Not Strategic

A Skill is an automated sequence of Operations. It answers the question **"how to perform THIS procedure."** But the question **"WHICH procedure to choose"** is Knowledge generated by principles.

An analogy from ML (MIM.D.007): a principle is the inductive bias of an architecture; a Skill is the trained weights. Without the right bias, no amount of data helps.

### 2. Skills Do Not Transfer Across Domains

A programmer's Skill is useless in surgery. An accountant's Skill is useless in design. Disciplinary Skills are context-bound.

Zero principles (ZP) and transdisciplines transfer by definition: ontology, logic, and Systems Thinking work everywhere. An agent who has mastered the transdisciplines learns a new Domain in weeks, not years.

### 3. A "Bucket of Skills" ≠ Integrated Capability

A team where each member can perform A, B, and C separately is not the same as a team that has mastered the integrated workflow A+B+C. The emergent value of Integration is invisible when you look only at Skills.

Mastery = Skills + coordination + judgment + method selection. The last three are not Skills — they are Knowledge.

### 4. Skills Do Not Generate New Skills

Principles are generative: new methods for new situations can be derived from them. Skills are executory: they work only in trained situations.

When a technological shift occurs — AI replaces functions, new professions emerge — an agent with principles creates new Roles. An agent with only Skills searches for retraining courses.

## The Generative Chain

```
ZP (think)  →  FPF (describe)  →  Pack (know)  →  DS (do)
     ↓                ↓                  ↓               ↓
 Transfer         Correctness       Domain language    Skills
 across           of descriptions   and methods        and programs
 domains
```

Each level **generates** the next. Without the upper levels, the lower levels either do not emerge or emerge slowly and fragility.

## What This Means for the IWE User

IWE is built according to this hierarchy:

| IWE Component | Principles level |
|---------------|-----------------|
| Methodology (FPF/SPF, protocols) | Level 0-1 |
| Exocortex (CLAUDE.md, memory/) | Level 1-2 |
| AI Roles (Strategist, Extractor) | Level 2-3 |
| Workspace (DS-strategy/) | Level 3 |
| Tools (Git, MCP, bot) | Level 3 |

An IWE user does not simply receive tools (Level 3). They receive **methodology** (Level 0-1) and **domain knowledge** (Level 2), which allow them to generate new Skills and adapt to change.

## Connection to Development Formats (МИМ)

| Level | What is learned | Format |
|-------|----------------|--------|
| ZP/FPF | Principles | Seminar (introduction) → Workshop (automation) |
| Pack | Domain knowledge | Seminar + Workshop |
| DS | Skills | Workshop → Residency (work result) |
