# Различения — hot-каркас (≤150 строк)

> Tier-A = суть + тест. Tier-B = краткая суть. Tier-C и полные тексты с источниками → `memory/distinctions-warm.md` (lazy = Read warm-файла). Правки сюда; авторские warm — туда.

## Tier-A (всегда)

- **Проблема ≠ Задача ≠ ФР ≠ Работа** (HD #24, DP.D.053). Ступор / метод известен / спецификация / физ.мир.
- **Inventory ≠ Progress** (Goldratt). Незавершённый РП/PR = замороженный капитал, не прогресс. Тест: «завершено физически → Throughput; в работе → Inventory».
- **Source-of-truth ≠ Производный.** Производный переиспользует ID из SoT, параллельно не нумерует. Тест: «копия-для-чтения или новый owner контента?»
- **Pack-знание ≠ Реализационное решение** (HD #29). Доменная истина → Pack. Технический выбор → DS.
- **Pack-тип ≠ DB-таблица ≠ DB-строка** (WP-251, FPF A.7). Класс (Episteme) / schema реализации / token. Связь — только явный маппинг `ECO.MAP.BIND.*`, не совпадение имён.
- **Процесс ≠ Артефакт-результат ≠ Артефакт-процесс** (2026-06-08). Процесс = деятельность без материального выхода, сам по себе ≠ РП. Тест: «готовый результат или чистая деятельность?» Конвейер держат и модернизируют, не закрывают как РП.
- **Скилл ≠ SKILL.md ≠ Роль ≠ Протокол.** Способность / спецификация-контракт / функциональное место / ритуал ОРЗ. Тест: скилл = умение играть, SKILL.md = партитура.
- **Микро-вопрос (P5) ≠ Ритуал WP Gate ≠ Choice-question.** yes/no запрещён / согласование РП обязательно / выбор альтернативы ОК.
- **WP-context ≠ Результат работы.** Context в DS-strategy; результат в целевом DS/Pack.
- **суть (ядро IWE-агента) ≠ Интерфейс** (WP-392). Суть = ориентация + знание + контекст пилота + память (через Aisystant MCP); интерфейс (бот, Hermes, Web UI, VS Code) лишь рендерит. Тест: «ответ зависит от того, ЧЕРЕЗ ЧТО спросили?» → суть протекла в интерфейс.
- **Агент ≠ Роль** (WP-350). Агент = {LLM, инструменты, промпт, персона, контекст}; роль = функциональное место (что делать, полномочия, домен). Тест: «уберу агента — роль остаётся?» Специализированный агент может зваться именем роли, универсальный — нет.
- **Персона ≠ Память ≠ Контекст** (HD #27, DP.D.052 v2). Критерий writer + owner: Персона = Ory anchor + Git declarations + Neon refs; Память = Neon (Observed + Derived); Контекст/Проекция = runtime ephemeral.
- **Универсальное руководство ≠ Персональное руководство** (WP-300). Оба — проекции Pack PD. Универсальное — метод для всех, без RCS; персональное — под пилота (ступень + bottleneck + домен). Не переписывать друг через друга.
- **Текущее ≠ Важное ≠ Срочное (ТВС)** (СС 14 июня). Текущее = штатная работа конвейера; срочное = угроза ОСТАНОВКИ конвейера; важное = поднятие нормы. Дедлайн ортогонален всем трём. Тест: «штатный выпуск / вернуть остановленный / поднять норму?»

## Tier-B (краткая суть; источники → warm)

- **LLM-в-центре ≠ ядро-в-центре** → детерм. ядро управляет итерациями; LLM — при неопределённости
- **Рефлекс ≠ интеллект** → lookup по сигнатуре признаков; при N повторах — кристаллизация
- **Решить задачу ≠ кристаллизовать класс** → разовый ответ vs исполняемый артефакт без LLM
- **OwnerIntegrity** → один факт — одно место; дублирование Pack↔DS = ошибка
- **Экзоскелет ≠ Автопилот** → усиление мышления vs LLM-автономия; тест: объяснишь без «ИИ подсказал»?
- **Скрипт ≠ Агент** → фиксированный flow vs LLM выбирает шаг; тест: уберу LLM — сломается?
- **settings.json ≠ settings.local.json** → проектный vs персональный
- **Model Tiering** → trivial → Haiku; closed-loop → Sonnet; open-loop/problem-framing → Opus
- **Терминальный fail ≠ Transient fail** → терминальный: mark_done; transient: retry
- **HOT horizon: hard limit 150 lines** → HOT-файл >150 строк → split или demote
- **Дедлайн ≠ Срочность** → дедлайн ортогонален ТВС; срочность = только угроза остановки
- **Удержание нормы ≠ Поднятие нормы** → управление держит норму; развитие перестраивает на лучшую
- **Доставляемый артефакт ≠ Сервис** → артефакт копируется к пользователю; сервис = один инстанс
- **Платформа ≠ IWE-помощник** → платформа первична, assistant-agnostic; IWE = один из клиентов
- **Pack ≠ Программа развития** → поле/домен (онтология) vs курс развития роли
- **Носитель ≠ Персона ≠ Декларация Персоны** → человек vs entity-в-IWE vs Git-snapshot
- **Состояние объекта ≠ Стадия процесса** → кто объект есть vs шаг воздействия на него
- **Vault ≠ Pack ≠ Файловая система** → хранилище с интерфейсом vs онтологический тип
- **Нумерация шагов протокола** → буквенные суффиксы уникальны в файле; вставка → дробный формат
- **Профиль ошибок агента ≠ DayPlan** → для агента (инжекция) vs для пилота (артефакт)
- **DRR-запись ≠ Pack-текст** → решение = owner (SoT); правку решения начинать с DR-записи
- **Гейт ≠ Демонстрационный порядок мантры** → [[gate]] блокирует, [[narrative]] skippable; Close проверяет набор gate, не линейность
